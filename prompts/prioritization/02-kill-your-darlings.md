# "Kill Your Darlings" Exercise

**Role:** Both

## Prompt

```
We have too many features/ideas on the table and need to cut scope. Here's what we're considering:

[List everything on the table]

Our constraints: [timeline, team size, dependencies]
Our core user need: [the one thing we must solve]

Force-rank these by asking:
1. If we could only ship ONE thing, which would it be? Why?
2. For everything else: what happens to the user if we don't build this? (If the answer is "not much," cut it)
3. Which items are we keeping because we're attached, not because users need them?
4. What's the minimum viable version that actually solves the core need?

Be honest with me. I need someone to help me let go.
```
