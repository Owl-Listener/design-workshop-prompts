# Run an Affinity Mapping Exercise

**Role:** Facilitator

## Prompt

```
I have [number] research observations from [interviews/usability tests/surveys]:

[Paste your observations, one per line]

Help me run an affinity mapping exercise:
1. Group these into clusters based on underlying themes (not surface similarity)
2. Name each cluster with an insight statement, not a topic label (e.g., "Users create workarounds because the main flow assumes expertise" not "Workarounds")
3. Identify which clusters have the most observations (signal strength)
4. Flag any outliers that don't fit a cluster but feel important
5. Suggest which 3 clusters the team should focus on first and why
```
