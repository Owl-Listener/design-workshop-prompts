# Prompt Pack for Design Workshops

36 ready-to-paste prompts for running better design workshops with any LLM.

Organized by activity. Labeled by role. Copy, paste, go.

**This works with any LLM** — ChatGPT, Claude, Gemini, Copilot, or whatever your team has access to. No plugins, no setup, no special tools required.

## What's inside

| Category | Prompts | What it covers |
|---|---|---|
| [Stakeholder Interviews](prompts/stakeholder-interviews/) | 5 | Interview scripts, debrief synthesis, cross-interview comparison, design principles |
| [Design Sprints](prompts/design-sprints/) | 6 | Challenge framing, HMW notes, sketching, test scripts, unsticking rooms, retros |
| [Critique Sessions](prompts/critique-sessions/) | 6 | Critique setup, structured feedback, synthesis, self-challenge, silent critique, decoding vague feedback |
| [Research Synthesis](prompts/research-synthesis/) | 6 | Affinity mapping, insight statements, journey maps, gap analysis, readouts, personas |
| [Prioritization](prompts/prioritization/) | 6 | Impact-effort matrices, scope cutting, MoSCoW, making a case, resolving deadlocks, evaluating trade-offs |
| [Roadmap Planning](prompts/roadmap-planning/) | 6 | Now/Next/Later, stress-testing, leadership translation, proposals, quarterly planning, narratives |

Every prompt is labeled:

- **Facilitator** — You're running the workshop
- **Participant** — You're contributing or preparing
- **Both** — Works for either role

## How to use

1. Pick a category and a prompt
2. Open any LLM (ChatGPT, Claude, Gemini, etc.)
3. Copy the prompt from the markdown file
4. Replace the `[bracketed sections]` with your specifics
5. Use the output as a starting point, not a final answer

**Pro tip:** Chain prompts together. Run a stakeholder interview synthesis, then feed the output into a design principles prompt.

## Interactive version

Open [`prompt-pack.html`](prompt-pack.html) in your browser for a filterable, copy-to-clipboard version with all 36 prompts. Works offline — just download and open.

## The philosophy

These prompts aren't about replacing design thinking with AI. They're about removing the blank-page problem so you can spend your energy on the actual design work.

The best prompt is one you modify. If something doesn't fit your context, change it. These are starting points, not scripts.

## Contributing

2. Add or edit prompts in the relevant `prompts/` folder
3. Follow the existing format: title, role label, prompt in a code block, optional tips
4. Open a PR with a brief description of what the prompt does and when to use it
